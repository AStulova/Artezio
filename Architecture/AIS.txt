В магазине оптики персонал должен иметь доступ к данным, которые  
содержат в себе информацию о товарах, клиентах, заказах и т.п.

Для магазина оптики подойдет трехуровневая архитектура. 
Это обусловленно тем, что:
1) Трехуровневая архитектура обеспечивает взаимодействие с пользователями с помощью графического 
интерфейса, что очень важно для удобной и простой работы пользователей с базой данных.
2) Такая архитектура обеспечивает изолированность уровней системы друг от друга, что обеспечивает
гибкость системы, т.е. возможность изменения каждого компонента с появлением новых требований.
3) Также существет возможность расширения функционала системы.

Трехуровневая архитектура состоит из следующих компонентов:
1) Сервер приложений 
2) Основное промежуточное ПО доступа к БД
3) Сервер баз данных

Сервер приложений получает запрос от пользователя, затем обрабатывает запрос пользователя с 
помощью соответствующего программного обеспечения. Основное промежуточное ПО осуществляет 
обращение к серверу БД в виде SQL-запроса. Сервер БД получает данные и предоставляет их серверу
приложений, а сервер приложений отправляет их пользователю. Данные предоставляются пользователю 
с помощью графического интерфейса.

Производительность:
1) Трехуровневая архитектура обеспечивает высокоскоростные сетевые соединения между серверами.
2) Возможнность выполнения запроса на разных серверах.
3) Трёхуровневая архитектура обеспечивает большую масштабируемость.

Безопасность:
1) Благодаря изолированности уровней системы клиент не имеет прямого доступа к серверу БД.
2) Трехуровневая архитектура осуществляет проверки безопасности на каждом уровне.

Надежность: 
1) Отказоустойчивость контуров на каждом слое за счет резервирования серверов.
2) Большое количество элементов в ценпочке увеличивает вероятность выхода из строя всей системы
в целом.
3) Возможность кэшировать ответы от сервера БД и использовать во время недоступности сервера БД.
